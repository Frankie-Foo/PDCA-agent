## 一、推荐：Docker 一键（配置已内置）

```bash
cd /opt/PDCA-agent
git pull origin main
bash pdca-workbench/scripts/docker_up.sh
```

或：

```bash
cd /opt/PDCA-agent/pdca-workbench
docker compose up -d --build
```

### 自动完成

- 挂载经营首页：`data_platform/data_role_pdca_mvp` → 容器 `/mvp`
- 使用仓库内 `pdca-workbench/env.docker`（库、密钥、`hybrid` 已写好）
- 强制 `PDCA_MVP_ROOT=/mvp`，避免「经营首页不可用」

### 首次无仓库

```bash
cd /opt
git clone https://github.com/Frankie-Foo/PDCA-agent.git
cd PDCA-agent
bash pdca-workbench/scripts/docker_up.sh
```

## 二、登录

| 方式 | 说明 |
|------|------|
| 本地账号 | 由管理员面板创建，或使用一次性 `PDCA_BOOTSTRAP_ADMIN_*` 初始化 |
| VPS | 登录页「VPS 一键进入」（服务器需 `vertu login`） |

认证模式：`PDCA_AUTH_MODE=hybrid`（本地 + VPS 均可）

---

## 三、环境变量说明

| 来源 | 何时用 |
|------|--------|
| **仓库 `pdca-workbench/env.docker`** | Docker 默认，**一般不用手改** |
| 可选 `.env` | 覆盖个别项；compose 仍强制 `/mvp`、`/repo` |

关键项：

```env
PDCA_AUTH_MODE=hybrid
PDCA_MVP_ROOT=/mvp          # Docker 内固定
PDCA_REPO_ROOT=/repo        # Docker 内固定
PDCA_SECURE_COOKIES=1
PDCA_DATABASE_URL=postgresql+psycopg2://...@10.100.0.176:5432/pdca
```

## 四、「经营首页不可用」

原因：容器/进程读不到 `home_dashboard/index.html`。

处理：

```bash
cd /opt/PDCA-agent && git pull origin main
bash pdca-workbench/scripts/docker_up.sh
```

确认宿主机有：

```bash
ls /opt/PDCA-agent/data_platform/data_role_pdca_mvp/modules/home_dashboard/index.html
```

## 五、常用命令

```bash
cd /opt/PDCA-agent/pdca-workbench
docker compose logs -f pdca-app
docker compose restart
docker compose down && docker compose up -d --build
curl http://127.0.0.1:8767/health
```

## 六、不用 Docker 时

```bash
cd /opt/PDCA-agent
git pull origin main
bash pdca-workbench/scripts/deploy_internal.sh
```

脚本会自动校正 `PDCA_MVP_ROOT` / `PDCA_REPO_ROOT` 并重启 systemd。

## 七、前置条件

- 部署机 IP 已加入 PG `10.100.0.176:5432` 白名单
- 已装 Docker（推荐）或 Python 3.12
- 完整克隆本仓库（不要只拷 `pdca-workbench`）
