# PDCA 工作台 · 运维部署手册

## 概览

将门店五件套录入系统部署到公网，供各国经销商使用。

- 入口页面：`https://你的域名/walkin-submit`
- 数据库：**云端 PostgreSQL 已就绪**（10.100.0.176:5432/pdca），表结构、门店数据、经销商账号均已初始化，直接用
- 后端：Python FastAPI，跑在 8767 端口，由 Nginx 反代

---

## 一、服务器要求

| 项目 | 要求 |
|---|---|
| OS | Ubuntu 22.04（推荐）或其他 Linux |
| Python | 3.12 |
| Nginx | 任意新版 |
| 端口 | 公网开放 80、443；8767 仅内网 Nginx 访问 |
| 到云端 DB | 服务器 IP 需加入 10.100.0.176 的 PostgreSQL 白名单（5432 端口）|

---

## 二、上传代码

```bash
# 将 pdca-workbench 目录上传到服务器
scp -r ./pdca-workbench user@YOUR_SERVER_IP:/opt/pdca-workbench
```

### 可选：上传 MVP 看板模块

> 不上传也能正常使用门店五件套录入功能；只有客流看板、会议中心等高级模块需要此目录。

```bash
# 将 MVP 模块目录上传到服务器
scp -r ./data_platform/data_role_pdca_mvp user@YOUR_SERVER_IP:/opt/data_role_pdca_mvp
```

然后在 `.env` 中添加：
```env
PDCA_MVP_ROOT=/opt/data_role_pdca_mvp
```

## 三、配置 .env

在服务器 `/opt/pdca-workbench/` 目录下创建 `.env`，内容如下：

```env
PDCA_HOST=0.0.0.0
PDCA_WORKBENCH_PORT=8767

# 用 openssl rand -hex 32 生成，不能用默认值
PDCA_SECRET_KEY=在这里填32位随机字符串

# 云端数据库，已有数据，不要改
PDCA_DATABASE_URL=postgresql+psycopg2://postgres:H1UaJoeo-aSF-zpM6V-0ARP@10.100.0.176:5432/pdca

# 对外部署必须是 local
PDCA_AUTH_MODE=local

# Nginx 做了 HTTPS 终止后必须是 1
PDCA_SECURE_COOKIES=1

PDCA_LOG_LEVEL=INFO
```

生成 SECRET_KEY：
```bash
openssl rand -hex 32
```

## 四、安装依赖并启动服务

```bash
cd /opt/pdca-workbench
pip install -r requirements.txt
python run.py
```

服务正常启动后会输出：
```
INFO:     Application startup complete.
```

验证是否可访问：
```bash
curl http://127.0.0.1:8767/health
# 返回 {"status":"ok"} 即正常
```

## 五、配置 systemd 开机自启

```bash
cat > /etc/systemd/system/pdca.service << 'EOF'
[Unit]
Description=PDCA Workbench
After=network.target

[Service]
User=root
WorkingDirectory=/opt/pdca-workbench
ExecStart=/usr/bin/python3 run.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable pdca
systemctl start pdca
```

## 六、配置 Nginx + HTTPS

申请证书（Let's Encrypt）：
```bash
apt install certbot python3-certbot-nginx -y
certbot --nginx -d 你的域名
```

创建 Nginx 配置 `/etc/nginx/sites-available/pdca`：

```nginx
server {
    listen 80;
    server_name 你的域名;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name 你的域名;

    ssl_certificate     /etc/letsencrypt/live/你的域名/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/你的域名/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;

    client_max_body_size 20m;

    location / {
        proxy_pass         http://127.0.0.1:8767;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
    }
}
```

```bash
ln -s /etc/nginx/sites-available/pdca /etc/nginx/sites-enabled/pdca
nginx -t && systemctl reload nginx
```

## 八、日常运维

**查看日志**
```bash
journalctl -u pdca -f
```

**重启服务**
```bash
systemctl restart pdca
```

**更新代码**
```bash
cd /opt/pdca-workbench
# 上传新代码后
systemctl restart pdca
```

**重置某个经销商密码**（admin 账号登录后台操作）
```
https://你的域名/admin-panel
```
用 admin 账号登录 → 用户管理 → 找到对应账号 → 重置密码
